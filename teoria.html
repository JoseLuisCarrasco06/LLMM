1.Elementos mal anidados

El contenido de <president> y <secretary> incluye otro elemento (<extension>) sin cerrarse correctamente. XML no permite que una etiqueta comience dentro de otra y se cierre fuera.

2. Etiquetas mal cerradas

El orden de cierre está invertido:

xml

<president>Bill Gates
<extension>...</president></extension>

Eso es incorrecto.

3. Contenido mixto mal estructurado

Si quieres incluir el nombre + una subetiqueta como <extension>, lo correcto es separarlos: el nombre en una etiqueta, y la extensión en otra dentro del mismo nivel.

<phonebook>
  <company>
    <cname>Microsoft</cname>

      <name>Bill Gates</name>
      <extension>09-9990011</extension>
    </president>
    <secretary>
      <name>Katharine Finch</name>
      <extension>09-9990012</extension>
    </secretary>

EJERCICIO 7:

a. ¿Cómo se representan los comentarios en XML? (sección 2.5)

Los comentarios en XML se escriben así:

<!-- Este es un comentario -->

Es decir, los comentarios están delimitados por <!-- al inicio y --> al final, y pueden contener cualquier texto excepto la secuencia -- dentro del comentario.

b. ¿Puede un atributo repetirse en el mismo elemento? (sección 3.1)

No, un atributo no puede repetirse dentro del mismo elemento. Cada nombre de atributo debe ser único para ese elemento. Por ejemplo, esto es incorrecto:

<elemento atributo="valor1" atributo="valor2">...</elemento>

Debe evitarse, porque es un error de sintaxis XML.

c. ¿Dónde se indica que los nombres de los elementos en las etiquetas de apertura y cierre deben coincidir?

Esto está especificado en la sección que define la well-formedness (formación correcta) del documento XML. Es decir, para que un documento XML sea válido, los nombres de los elementos deben coincidir en la etiqueta de apertura y en la de cierre.

Por ejemplo:

<item>Contenido</item> <!-- Correcto -->

<item>Contenido</itm> <!-- Incorrecto -->
